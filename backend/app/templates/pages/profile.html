Backup

Un backup est une archive .zip contenant l'intégralité de vos données : documents et leurs fichiers, correspondants, types, tags, rappels, sources Gmail et logs d'import. Il ne contient pas votre compte utilisateur (mot de passe, email). Vous pouvez le restaurer sur n'importe quelle instance ProCompta.

Télécharger

Télécharge un fichier .zip sur votre appareil

Attention - opération irréversible

Toutes les données actuelles (documents, correspondants, tags, notifications…) seront remplacées par le contenu du backup. Confirmez votre mot de passe pour continuer.

  • Correspondants, types de documents, tags

  • Votre compte utilisateur est conservé

Réinitialisation complète

À exécuter depuis un terminal, dans le dossier du projet.

Ce que ces commandes font :

  • Arrête et supprime les containers procompta-api, procompta-db, procompta-caddy
  • Supprime les images Docker construites
  • Supprime les données PostgreSQL (postgres_data/) et les fichiers (storage/)
  • Le dossier backups/ n'est pas touché
Commandes à exécuter 
docker-compose down --rmi all
rm -rf postgres_data storage

Après suppression, relancez avec docker-compose up -d --build pour repartir de zéro.
